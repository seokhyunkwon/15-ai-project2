# 대구 시내버스 정류소 이용 수요 및 노선 공급 불균형 분석

## 1. 프로젝트 소개

이 프로젝트는 대구 시내버스 정류소별 승하차 인원, 시간대별 이용 패턴, 경유 노선 수, 월별 이용량, 날씨 데이터를 함께 분석하는 Streamlit 기반 데이터 분석 웹앱입니다.

단순히 이용객이 많은 정류소 순위를 보여주는 것이 아니라, 이용 수요에 비해 경유 노선 수가 상대적으로 적은 정류소를 찾아 “추가 검토 후보”로 제시하는 것이 핵심입니다.

## 2. 프로젝트 목적

- 대구에서 이용객이 많은 정류소 확인
- 출근 시간과 퇴근 시간에 수요가 집중되는 정류소 확인
- 구·군별 버스 이용량 차이 비교
- 경유 노선 수와 승차 인원의 관계 분석
- 이용객 수에 비해 경유 노선 수가 적은 정류소 후보 탐색
- 정류소를 출근형, 퇴근형, 출퇴근형, 생활형, 저이용형으로 분류
- 월별 이용량 추세와 전년 동월 대비 증감 확인
- 월별 날씨 변수와 버스 이용량의 상관관계 탐색

## 3. 데이터 구성

원본 CSV는 `data/raw` 폴더에 넣습니다.

현재 프로젝트에서 사용하는 주요 데이터는 다음과 같습니다.

- 정류소별 시간대별 승차·하차 인원 데이터
- 정류소 위치정보 데이터
- 정류소별 경유 노선 수 또는 경유 노선 정보
- 정류소별 월별 이용자 수 데이터
- 대구 월별 날씨 데이터

CSV 인코딩은 파일마다 다를 수 있으므로 `utf-8`, `utf-8-sig`, `cp949`, `euc-kr`을 순서대로 시도하여 읽습니다.

실제 파일명과 컬럼명이 달라도 앱이 비슷한 의미의 컬럼을 자동 탐지합니다. 단, 실제 데이터에 없는 컬럼을 임의로 만들어 분석하지 않습니다.

## 4. 데이터 정제 방식

원본 CSV를 Streamlit에서 바로 분석하지 않고, 먼저 `prepare_data.py`로 정제합니다.

정제 과정은 다음 순서로 진행됩니다.

1. CSV 파일 자동 탐색
2. 인코딩 자동 판별
3. 파일별 행 수, 열 수, 컬럼명, 결측값 등 점검
4. 컬럼명 앞뒤 공백 제거
5. 쉼표가 포함된 숫자 문자열 정리
6. 승차 인원, 하차 인원, 노선 수 등 숫자형 변환
7. 날짜, 연도, 월 컬럼 정리
8. 시간대 컬럼을 long 형식으로 변환
9. 정류소 ID 또는 정류소명 기반 병합 키 생성
10. 위치정보, 월별 이용량, 시간대별 이용량, 날씨 데이터 정리
11. 중복 데이터 제거 및 음수 인원 등 이상값 처리
12. 분석에 필요한 파생변수 생성

시간대별 데이터가 `05시`, `06시`, `07시`처럼 여러 컬럼으로 되어 있으면 `pandas.melt()`를 사용하여 다음과 같은 구조로 바꿉니다.

```text
정류소명 | 구·군 | 시간대 | 승차 인원 | 하차 인원
```

이렇게 바꾸면 Streamlit에서 시간대별 선그래프, 히트맵, 정류소별 패턴 비교를 쉽게 만들 수 있습니다.

## 5. 정제 데이터 저장 및 Streamlit 로드 구조

전처리 명령을 실행하면 `outputs/processed` 폴더에 정제 CSV가 저장됩니다.

```powershell
python prepare_data.py
```

주요 출력 파일은 다음과 같습니다.

```text
outputs/processed/
├─ data_check_report.csv
├─ column_mapping.csv
├─ stop_summary.csv
├─ hourly_summary.csv
├─ monthly_summary.csv
├─ imbalance_candidates.csv
├─ weather_monthly.csv
├─ bus_weather_monthly_merged.csv
├─ weather_correlation_results.csv
└─ analysis_summary.md
```

Streamlit 앱은 원본 CSV를 매번 다시 정제하지 않고, `outputs/processed`에 저장된 정제 CSV를 불러옵니다.

앱 내부에서는 `st.cache_data`와 `st.session_state`를 사용하여 한 번 불러온 데이터를 세션에 저장합니다. 따라서 탭을 이동하거나 필터를 변경할 때마다 CSV를 다시 읽지 않아도 됩니다.

원본 CSV를 교체하거나 새 파일을 추가한 경우에는 다음 순서로 다시 실행합니다.

```powershell
python prepare_data.py
streamlit run app.py
```

앱 실행 중에는 사이드바의 `정제 데이터 다시 불러오기` 버튼으로 저장된 정제 CSV를 다시 로드할 수 있습니다.

## 6. 주요 파생변수

분석을 위해 다음 파생변수를 계산합니다.

- 전체 이용객 = 승차 인원 + 하차 인원
- 노선당 승차 인원 = 승차 인원 / 경유 노선 수
- 출근 시간 승차 인원 = 7시부터 9시까지 승차 인원
- 퇴근 시간 승차 인원 = 17시부터 20시까지 승차 인원
- 출근 집중도 = 출근 시간 승차 인원 / 전체 승차 인원
- 퇴근 집중도 = 퇴근 시간 승차 인원 / 전체 승차 인원
- 최대 혼잡 시간대 = 승차 인원이 가장 많은 시간대
- 승하차 차이 = 승차 인원 - 하차 인원
- 승하차 불균형 지수 = abs(승차 인원 - 하차 인원) / 전체 이용객
- 전년 동월 대비 증감률 = 현재 월 이용량과 전년 같은 달 이용량 비교

경유 노선 수가 0이거나 결측인 경우에는 나눗셈 오류가 생기지 않도록 처리합니다.

## 7. 정류소 유형 분류

정류소는 시간대별 이용 패턴을 기준으로 다음 유형으로 분류합니다.

- 출근형: 출근 시간 집중도가 높고 퇴근 시간 집중도보다 큰 정류소
- 퇴근형: 퇴근 시간 집중도가 높고 출근 시간 집중도보다 큰 정류소
- 출퇴근형: 출근과 퇴근 시간 집중도가 모두 높은 정류소
- 생활형: 특정 시간대에 과도하게 몰리지 않고 비교적 고르게 이용되는 정류소
- 저이용형: 전체 승차 인원이 하위 25% 수준인 정류소

추가로 K-means 군집화를 사용하여 시간대별 이용 비율이 비슷한 정류소끼리 묶을 수 있습니다.

K-means 군집화에서 사용하는 값은 실제 인원 수가 아니라, 각 정류소의 전체 승차 인원 중 특정 시간대가 차지하는 비율입니다. 예를 들어 `0.0043`은 0.43%, 즉 약 0.4%를 의미합니다.

## 8. 상관계수 해석

앱에서는 Pearson 상관계수와 Spearman 상관계수를 사용합니다.

### Pearson 상관계수

Pearson 상관계수는 두 변수가 선형적으로 함께 움직이는 정도를 의미합니다.

예를 들어 경유 노선 수가 증가할수록 승차 인원도 직선적인 관계로 증가하는지 확인할 때 사용합니다.

### Spearman 상관계수

Spearman 상관계수는 두 변수의 순위가 함께 움직이는 정도를 의미합니다.

값 자체가 직선 관계가 아니더라도, 노선 수가 많은 정류소가 대체로 이용객도 많은지처럼 순위 기반 경향을 확인할 수 있습니다.

### 해석 기준

- 0에 가까움: 관계가 약함
- 0.3 전후: 약한 상관
- 0.5 전후: 보통 수준의 상관
- 0.7 이상: 강한 상관

상관관계는 인과관계를 의미하지 않습니다. 예를 들어 “강수량 때문에 이용객이 감소했다”가 아니라 “강수량이 높은 달에 이용량이 감소하는 경향이 관찰되었다”라고 해석합니다.

## 9. 탭별 화면 구성

### 전체 현황

전체 승차 인원, 전체 하차 인원, 분석 대상 정류소 수, 가장 이용객이 많은 정류소, 가장 혼잡한 시간대, 노선당 승차 인원이 높은 정류소를 요약합니다.

정류소 TOP N, 구·군별 승차 인원, 시간대별 승차 인원, 정류소 유형별 비율을 통해 대구 버스 이용 현황을 전체적으로 파악합니다.

### 시간대별 분석

시간대별 전체 승차 인원, 승차·하차 비교, 정류소 × 시간대 히트맵, 구·군 × 시간대 히트맵을 보여줍니다.

출근 시간과 퇴근 시간에 어느 정류소와 어느 구·군에서 수요가 집중되는지 확인하는 탭입니다.

### 정류소별 분석

사용자가 정류소 하나를 선택하면 해당 정류소의 전체 승차 인원, 하차 인원, 경유 노선 수, 노선당 승차 인원, 최대 혼잡 시간대, 출근·퇴근 집중도를 보여줍니다.

개별 정류소가 전체 평균 또는 같은 구·군 평균과 비교해 어떤 특징을 가지는지 확인합니다.

### 수요·공급 불균형 분석

경유 노선 수와 전체 승차 인원의 관계를 산점도로 보여줍니다.

전체 승차 인원이 높고, 경유 노선 수가 상대적으로 적으며, 노선당 승차 인원이 높은 정류소를 추가 검토 후보로 제시합니다.

이 분석은 실제 노선 부족을 확정하는 것이 아니라, 추가 검토가 필요한 후보 정류소를 좁히는 목적입니다.

### 지도 분석

정류소 위치정보가 있는 경우 지도 위에 정류소를 표시합니다.

정류소 위치와 이용량 분포를 함께 보면서 특정 지역에 수요가 집중되는지 확인할 수 있습니다.

### 장기 추세 분석

월별 이용량 데이터를 사용하여 장기 추세를 확인합니다.

현재 앱에서는 2026년 월별 승차 인원을 2025년 같은 달과 비교하여 전년 동월 대비 증감률을 보여줍니다.

### 날씨와 버스 이용

월별 날씨 데이터와 월별 버스 이용량을 결합하여 분석합니다.

중요한 점은 날씨 값을 정류소별 행에 반복해서 붙인 상태로 상관분석하지 않는다는 것입니다. 대구 전체 날씨와 버스 이용량을 비교할 때는 버스 이용량을 먼저 월별 전체 값으로 집계한 뒤, 월별 날씨 데이터와 한 행씩 결합합니다.

이 탭에서는 다음을 확인합니다.

- 월별 버스 승차 인원 추세
- 월별 기온 추세
- 월별 버스 이용량과 날씨 추세 비교
- 기온과 승차 인원의 산점도
- 강수량과 승차 인원의 산점도
- 계절별 이용량 비교
- 날씨 변수별 Pearson·Spearman 상관계수

### 데이터 해석 참고사항

분석 결과를 해석할 때 함께 고려해야 하는 조건을 정리합니다.

하차 태그 누락 가능성, 현금 승차 제외 가능성, 실제 배차 횟수와 경유 노선 수의 차이, 배차 간격과 차량 크기 정보 부재, 공휴일·방학·지역 행사·노선 개편 같은 외부 요인을 함께 확인합니다.

## 13. 발표 시 강조할 내용

발표에서는 다음 흐름을 강조하면 좋습니다.

1. 원본 CSV를 그대로 쓰지 않고 pandas로 먼저 정제했다.
2. 정제된 CSV를 `outputs/processed`에 저장하고 Streamlit은 이 파일을 불러온다.
3. 단순 TOP 10이 아니라 수요와 공급의 균형을 함께 본다.
4. 승차 인원, 경유 노선 수, 노선당 승차 인원을 함께 사용해 추가 검토 후보를 찾는다.
5. 정류소 유형 분류를 통해 출근형, 퇴근형, 생활형 등 이용 패턴을 해석한다.
6. 날씨 분석은 월별 버스 이용량과 월별 날씨 데이터를 같은 시간 단위로 맞춘 뒤 수행한다.
7. 상관계수는 경향을 보여주는 지표이며 인과관계로 단정하지 않는다.
8. 분석 결과는 노선 부족 확정이 아니라 정책 검토를 위한 후보 탐색 자료로 활용한다.

앱 실행 후 `outputs/processed/analysis_summary.md`에서 자동 생성된 분석 요약을 확인할 수 있습니다.

정적 그래프는 사이드바에서 `정적 그래프 표시 여부`를 켠 뒤 `정적 그래프 PNG 저장` 버튼을 누르면 `outputs/figures` 폴더에 저장됩니다.

## 14. 데이터 해석 참고사항

- 하차 태그를 하지 않은 승객이 있을 수 있어 하차 인원은 보조 지표로 함께 확인합니다.
- 현금 승차가 데이터에 포함되지 않을 수 있어 승차 인원 해석 시 집계 범위를 함께 확인합니다.
- 경유 노선 수와 실제 배차 횟수는 다를 수 있어 노선 수는 공급 규모의 1차 지표로 사용합니다.
- 버스 배차 간격과 차량 크기 자료가 추가되면 후보 정류소의 검토 우선순위를 더 정교하게 만들 수 있습니다.
- 이용객 수는 실제 차량 내부 혼잡도를 직접 확정하기보다 혼잡 가능성을 살피는 출발점으로 사용합니다.
- 특정 정류소의 높은 이용량은 주변 학교, 상권, 병원, 환승센터 등 지역 맥락과 함께 해석해야 합니다.
- 데이터 수집 기간에 따라 계절성과 일시적 이벤트가 함께 나타날 수 있습니다.
- 공휴일과 방학, 노선 개편, 지역 행사, 유가 변화, 코로나19 등의 외부 요인도 이용량 해석 조건으로 함께 고려해야 합니다.
- 월별 자료는 개별 강수일의 즉각적 영향보다 월 단위 흐름을 확인하는 데 적합합니다.
