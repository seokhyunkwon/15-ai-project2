# 대구 시내버스 정류소 이용 수요 및 노선 공급 불균형 분석

## 1. 프로젝트 소개

이 프로젝트는 Streamlit을 이용해 대구 시내버스 정류소별 승하차 수요와 경유 노선 수의 관계를 분석하는 데이터 분석 앱입니다.

단순히 이용객이 많은 정류소를 보여주는 것을 넘어, 이용 수요 대비 노선 공급이 상대적으로 부족해 보이는 정류소를 “추가 검토 후보”로 탐색합니다.

## 2. 프로젝트 목적

- 대구에서 이용객이 많은 정류소 확인
- 출근 시간과 퇴근 시간에 수요가 집중되는 정류소 확인
- 구·군별 버스 이용량 비교
- 경유 노선 수와 승차 인원의 관계 분석
- 이용객 수에 비해 경유 노선 수가 적은 정류소 후보 탐색
- 정류소를 출근형, 퇴근형, 출퇴근형, 생활형, 저이용형으로 분류
- 월별 장기 데이터가 있을 경우 증가·감소 정류소 확인

## 3. 데이터 파일 준비 방법

CSV 파일을 `data` 폴더에 넣습니다.

예상 데이터는 다음과 같습니다.

- 정류소별 시간대별 승차·하차 인원 데이터
- 정류소 위치정보 데이터
- 정류소별 경유 노선 수 또는 경유 노선 정보
- 정류소별 월별 이용자 수 데이터
- 대구 일별 날씨 데이터

CSV 인코딩은 `utf-8`, `utf-8-sig`, `cp949`, `euc-kr`을 순서대로 시도합니다.

실제 파일명과 컬럼명이 달라도 앱이 비슷한 의미의 컬럼을 자동으로 탐지합니다. 단, 데이터에 없는 컬럼을 임의로 만들어 분석하지 않습니다.

날씨 데이터는 날짜 또는 일시, 평균기온, 최고기온, 최저기온, 일강수량, 평균습도, 평균풍속, 일조시간, 적설량과 비슷한 실제 컬럼만 사용합니다. 버스 장기 데이터가 월별이면 일별 날씨를 월별로 집계한 뒤 결합합니다.

```powershell

```

## 7. Streamlit 실행 방법

```powershell

```

브라우저가 자동으로 열리지 않으면 Streamlit이 출력하는 `http://localhost:8501` 주소를 브라우저에 입력합니다.

## 9. 주요 분석 내용

- 전체 승차 인원이 많은 정류소 TOP N
- 노선당 승차 인원이 높은 정류소 TOP N
- 구·군별 전체 승차 인원
- 시간대별 전체 승차 인원
- 출근 시간 및 퇴근 시간 이용객 TOP N
- 정류소 유형별 개수
- 경유 노선 수와 승차 인원의 상관관계
- 노선 수는 적지만 이용객이 많은 정류소
- 노선 수는 많지만 이용객이 적은 정류소
- 승차와 하차의 차이가 큰 정류소
- 장기 데이터가 있을 경우 이용량 증가율·감소율 TOP N
- 월별 날씨 지표와 버스 이용량의 Pearson·Spearman 상관관계
- 기온·강수량과 월별 승차 인원의 산점도
- 계절별 버스 이용량 비교

## 10. 데이터 한계

- 하차 태그를 하지 않은 승객이 있을 수 있어 실제 하차 인원과 차이가 날 수 있습니다.
- 현금 승차가 데이터에서 제외될 수 있습니다.
- 경유 노선 수가 많다고 실제 배차 횟수가 많은 것은 아닙니다.
- 버스 배차 간격과 차량 크기 데이터가 없습니다.
- 이용객 수만으로 실제 차량 내부 혼잡도를 확정할 수 없습니다.
- 특정 정류소의 이용량이 높은 이유는 주변 학교, 상권, 병원, 환승센터 등 추가 정보가 필요합니다.
- 분석 결과는 노선 부족을 확정하는 것이 아니라 추가 검토 후보를 찾는 것입니다.
- 데이터 수집 기간에 따라 계절성과 일시적 이벤트의 영향을 받을 수 있습니다.
- 공휴일과 방학은 월별 이용량에 영향을 줄 수 있지만 현재 분석에는 별도로 반영하지 않았습니다.
- 노선 개편, 지역 행사, 유가 변화, 코로나19 등의 외부 요인이 이용량에 영향을 줄 수 있습니다.
- 월별 자료에서는 개별 강수일의 즉각적인 영향을 확인하기 어렵습니다.

## 11. 발표 시 강조할 핵심 결과

앱 실행 후 `outputs/processed/analysis_summary.md`에 분석 요약이 저장됩니다.

요약 문서에는 다음 항목이 포함됩니다.

- 가장 이용객이 많은 정류소
- 가장 혼잡한 시간대
- 출근형 정류소 대표 사례
- 퇴근형 정류소 대표 사례
- 노선당 승차 인원이 높은 정류소
- 수요·공급 불균형 추가 검토 후보
- 구·군별 주요 특징
- 분석 결과의 한계
- 정책적 활용 가능성

정적 그래프는 사이드바에서 `정적 그래프 표시 여부`를 켠 뒤 `정적 그래프 PNG 저장` 버튼을 누르면 `outputs/figures` 폴더에 저장됩니다.
